425 /// Emits the IR computing the multi-sized tiling specification with two tile
426 /// sizes not exceeding `targetSize`, each divisible by `sizeDivisor`, such
427 /// that there exist numbers of tiles with these sizes that fully cover the
428 /// given iteration space `dimension` of the structured `op`.
429 ///
430 /// The computation is as follows:
431 ///
432 /// b = originalTripCount floordiv sizeDivisor
433 /// t = (targetSize + sizeDivisor - 1) floordiv sizeDivisor
434 /// d = (b + t - 1) floordiv t
435 /// s = (b floordiv d) * sizeDivisor
436 /// v = b % d
437 /// u = d - v
438 ///
439 /// where the tile sizes are `s` and `s` + `sizeDivisor`, and the numbers of
440 /// the corresponding tiles are `u` and `v`, respectively. Alternatively,
441 ///
442 /// s * u + (s + sizeDivisor) * v == original size,
443 /// where s mod sizeDivisor = 0.
444 ///
445 /// Expects all values to be positive. In some cases with the target tile size
446 /// sufficiently close to the dimension shape and non-unit divisor, it is
447 /// impossible to compute such sizes. If `emitAssertion` is set, also emit the
448 /// assertion that size computation succeeded.
449 ///
450 /// Returns the specification consisting of both tile values and the number of
451 /// tiles of each size.
